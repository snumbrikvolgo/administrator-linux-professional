# Задание 12: Первые шаги с Ansible

## Цель домашнего задания

Развернуть `nginx` на сервере с помощью Ansible. Настройка выполняется через playbook и Ansible-роль.

## Описание домашнего задания

Подготовить стенд на Vagrant как минимум с одним сервером. На этом сервере, используя Ansible, необходимо развернуть nginx со следующими условиями:

- необходимо использовать модуль `yum/apt`;
- конфигурационные файлы должны быть взяты из шаблона `jinja2` с переменными;
- после установки `nginx` должен быть в режиме enabled в `systemd`;
- должен быть использован `notify` для старта `nginx` после установки;
- сайт должен слушать на нестандартном порту -- 8080, для этого использовать переменные в Ansible.

## Стенд

Для создания виртуальной машины используется `Vagrantfile`.

В стенде создается одна виртуальная машина:

- имя хоста: `nginx`;
- box: `bento-ubuntu-24.04-local`;
- IP-адрес: `192.168.11.150`;
- оперативная память: `768 MB`;
- CPU: `1`;
- сеть: `private_network`.

## Основная логика работы

Для выполнения задания используется смешанная схема: Vagrant и VirtualBox запускаются из Windows, а Ansible запускается из WSL. Сначала Vagrant создает виртуальную машину nginx с private IP 192.168.11.150. Затем в WSL копируется приватный ключ Vagrant в домашний каталог пользователя, чтобы SSH мог использовать его с корректными правами. В inventory-файле Ansible явно указывается хост 192.168.11.150, пользователь vagrant и путь к скопированному ключу. После этого команда a`nsible -i staging/hosts nginx -m ping` проверяет, что управляющая машина может подключиться к серверу по SSH и выполнять на нем Ansible-модули. Успешный ответ `pong` означает, что стенд готов для запуска основного playbook, который устанавливает и настраивает `nginx`.

## Инструкция по запуску стенда

### Windows

Запустить виртуальную машину:

```bash
vagrant up
```

Проверить подключение к серверу:

```bash
vagrant ssh nginx
```

### WSL

Проверить доступность хоста через Ansible:

```bash
ansible -i staging/hosts nginx -m ping
```

Запустить playbook:

```bash
ansible-playbook -i staging/hosts playbook.yml
```

## Проверка

### WSL

Проверить работу nginx:

```bash
curl http://192.168.11.150:8080
```
### Windows

Проверить статус nginx:

```bash
vagrant ssh nginx
sudo systemctl status nginx
```

Проверить, включен ли nginx в автозагрузку:

```bash
$ sudo systemctl is-enabled nginx
enabled
```

Проверить, что nginx слушает порт `8080`:

```bash
sudo ss -tulpn | grep ':8080'
```
