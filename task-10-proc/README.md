# Задание 10: Работа с процессами

## Цель домашнего задания

Реализовать два конкурирующих за CPU процесса, запустить их с разными значениями `nice`, замерить время выполнения каждого процесса и сохранить лог выполнения.

## Описание домашнего задания

Необходимо написать bash-скрипт, который:

- запускает два CPU-bound процесса;
- ограничивает их выполнение одним CPU-ядром;
- задает процессам разные значения `nice`;
- измеряет время выполнения каждого процесса;
- выводит информацию о запущенных процессах;
- сохраняет результат работы в лог-файл.

## Основная логика скрипта

Скрипт запускает два одинаковых процесса.
Оба процесса закрепляются на одном CPU с помощью taskset -c "$CPU".
Затем процессам задаются разные приоритеты через `nice`: первый запускается с nice="$NICE_A", второй -- с nice="$NICE_B".
Каждый процесс генерирует поток данных размером `HASH_MB` мегабайт командой `dd if=/dev/zero bs=1M count="$HASH_MB"`.
Этот поток сразу передается в `sha256sum`, где вычисляется SHA-256 хеш. 

Перед началом вычислений скрипт фиксирует время старта.
После завершения фиксируется время окончания и считается длительность работы.
Оба процесса запускаются в фоне через &, их PID сохраняются в переменные.
Текущее состояние процессов выводится командой `ps`.
Затем основной скрипт ждет завершения процессов через `wait`.
В конце в консоль и лог выводится время выполнения каждого процесса.

В начале скрипта [nice.sh](nice.sh) заданы переменные:

```bash
HASH_MB="${HASH_MB:-2048}"  # размер буфера для вычисления хеша
CPU="${CPU:-0}" # выбор ядра процессора
NICE_A="${NICE_A:-0}" # приоритет процесса А
NICE_B="${NICE_B:-19}"  # приоритет процесса Б
LOG_FILE="${LOG_FILE:-./process_nice_$(date +%Y%m%d_%H%M%S).log}"
```


Сначала нужно выдать права на выполнение:

```bash
chmod +x process_nice_test.sh
```

Запуск со значениями по умолчанию:

```bash
./process_nice_test.sh
```

Запуск с другими переменными:

```bash
HASH_MB=4096 NICE_A=0 NICE_B=10 CPU=1  ./process_nice_test.sh
```

## Результат выполнения

В результате выполнения были запущены два процесса, которые выполняли одинаковую CPU-нагрузку.
Оба процесса были ограничены одним CPU-ядром с помощью команды `taskset`.

Первый процесс был запущен с обычным приоритетом:

```bash
nice=0
```

Второй процесс был запущен с пониженным приоритетом:

```bash
nice=19
```

Так как оба процесса конкурировали за один CPU, процесс с меньшим значением `nice` получил больше процессорного времени и завершился быстрее.
